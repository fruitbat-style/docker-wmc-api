CONTEXT
-------
Deploy the "Where's My Chai?" app (frontend + API + database) to a single EC2
instance running Docker Compose — the cheapest AWS option at ~$6-9/month.
A GitHub Actions pipeline auto-deploys on every push to main.

Repo: https://github.com/fruitbat-style/docker-wmc-api


================================================================================
  ARCHITECTURE
================================================================================

  Internet → EC2 t4g.micro ($6/mo)
             ├── Caddy (reverse proxy) → port 80
             │     ├── /api/*  →  API container (5232, internal)
             │     └── /*      →  Frontend static files (served by Caddy)
             ├── API container (.NET 10 runtime)
             ├── DB container (Postgres 17, internal only)
             └── pgdata volume (persistent on EBS)

All 4 containers (db, api, frontend build, caddy) run via docker-compose.prod.yml.
Database port is NOT exposed externally. Only Caddy on port 80 is public.

ESTIMATED COST:
  - EC2 t4g.micro:  ~$6/mo (free tier eligible year 1)
  - EBS 20GB gp3:   ~$1.60/mo
  - Elastic IP:     free while attached
  - TOTAL:          ~$8/mo

================================================================================
  HOW IT WORKS
================================================================================

DOCKER-COMPOSE.PROD.YML SERVICES:

  db:        Postgres 17, internal network only, healthcheck enabled,
             persistent pgdata volume

  api:       .NET 10 runtime (not SDK), production mode,
             auto-migrates DB on startup, seeds data if empty,
             no ports exposed (Caddy proxies internally)

  frontend:  Build-only container — runs npm build, outputs static files
             to a shared volume that Caddy reads

  caddy:     Reverse proxy on port 80
             - /api/* requests → forwarded to api:5232
             - /* requests → serves frontend static files with SPA fallback
             - To add HTTPS later: replace ":80" with "yourdomain.com" in Caddyfile


================================================================================
  CI/CD PIPELINE (.github/workflows/deploy.yml)
================================================================================

TRIGGER: Every push to the main branch

FLOW:
  1. test job:    Checkout → Setup .NET 10 → Run unit tests (62 tests)
  2. deploy job:  (only if tests pass) SSH into EC2 → git pull → rebuild → restart

GITHUB SECRETS REQUIRED:
  - EC2_HOST      EC2 public IP address
  - EC2_SSH_KEY   SSH private key for the ec2-user on the instance

PRODUCTION ENV VARS:
  Live in .env on the EC2 instance (NOT in git). Template:

    VITE_GOOGLE_MAPS_API_KEY=your-production-key
    POSTGRES_USER=wmc
    POSTGRES_PASSWORD=<strong-password-here>
    POSTGRES_DB=wmcdb


================================================================================
  EC2 ONE-TIME SETUP
================================================================================

Step 1: Launch EC2 instance
  - AMI: Amazon Linux 2023 (ARM/aarch64)
  - Instance type: t4g.micro
  - Storage: 20GB gp3
  - Security group: Allow inbound TCP 80 (HTTP) and 22 (SSH)
  - Allocate and associate an Elastic IP

Step 2: SSH in and install Docker

  sudo dnf update -y
  sudo dnf install -y docker git
  sudo systemctl enable docker && sudo systemctl start docker
  sudo usermod -aG docker ec2-user
  # Log out and back in for group change to take effect

Step 3: Install Docker Compose plugin

  sudo mkdir -p /usr/local/lib/docker/cli-plugins
  sudo curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-aarch64 \
    -o /usr/local/lib/docker/cli-plugins/docker-compose
  sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

Step 4: Clone repo and configure

  git clone https://github.com/fruitbat-style/docker-wmc-api.git docker-wmc
  cd docker-wmc
  cp .env.production.example .env
  # Edit .env with production values (strong password, production API key)

Step 5: Start the application

  docker compose -f docker-compose.prod.yml up -d --build

Step 6: Verify

  curl http://localhost/api/locations/filters
  # Should return JSON with flavors and product types


================================================================================
  COMMANDS REFERENCE
================================================================================

LOCAL DEV (existing):
  docker compose up --build           Start dev environment (hot reload)
  docker compose down                 Stop dev environment
  docker compose down -v              Stop and wipe database

PRODUCTION:
  docker compose -f docker-compose.prod.yml up -d --build    Start/rebuild
  docker compose -f docker-compose.prod.yml down             Stop
  docker compose -f docker-compose.prod.yml logs -f          View logs
  docker compose -f docker-compose.prod.yml logs -f api      API logs only
  docker compose -f docker-compose.prod.yml ps               Check status

GITHUB SECRETS SETUP:
  Go to repo → Settings → Secrets and variables → Actions → New repository secret
  Add: EC2_HOST = <your-ec2-elastic-ip>
  Add: EC2_SSH_KEY = <contents of your .pem private key>

================================================================================
  ADDING HTTPS LATER
================================================================================

When you have a domain:

  1. Point your domain's A record to the EC2 Elastic IP
  2. Edit Caddyfile: replace ":80" with "yourdomain.com"
  3. Add port 443 to docker-compose.prod.yml caddy service:
       ports:
         - "80:80"
         - "443:443"
  4. Add port 443 to EC2 security group inbound rules
  5. Rebuild: docker compose -f docker-compose.prod.yml up -d
  6. Caddy auto-provisions Let's Encrypt SSL certificate (free)
